Menú de navegación (Navbar)

¿Qué es y para qué sirve?

El menú es el “mapa rápido” del usuario: conecta las secciones principales del sitio y reduce la confusión. Un menú profesional prioriza claridad y consistencia sobre tener muchas opciones.

Tipos de menú (cuándo usar cada uno)
  • Horizontal (topbar): ideal para 4–7 opciones principales.
  • Hamburguesa: recomendado en móvil o cuando hay muchas secciones.
  • Sidebar: útil para paneles/admin o sitios con navegación profunda.
  • Dropdown: agrupa sub-secciones, pero úsalo con moderación.
  • Sticky: se queda visible al hacer scroll; útil si el sitio es largo.
Buenas prácticas (UX)
  • Prioriza tareas: lo más importante primero (no al final).
  • Nombres claros: “Servicios”, “Contacto”, “Precios” > “Soluciones integrales”.
  • Estado activo: indica dónde estás (en este proyecto se usa routerLinkActive).
  • Consistencia: mismo orden y etiquetas en todas las páginas.
  • Evita el ruido: si hay 10+ links, considera agrupar o usar buscador.
Accesibilidad (a11y) y teclado
  • Foco visible: al navegar con Tab, debe verse claramente dónde estás.
  • Targets grandes: links y botones clicables sin “micro-objetivos”.
  • Semántica: usa <nav> para navegación y texto descriptivo en enlaces.
  • Dropdowns: deben poder abrirse/cerrarse con teclado y tener buen orden de tabulación.
Checklist rápido del menú
  1. 5–7 opciones principales (o agrupadas si son más).
  2. “Activo” funcionando en cada ruta.
  3. Mobile: el menú no rompe el layout ni provoca scroll horizontal.
  4. Accesible: se puede usar con teclado y el foco se ve.
  5. Claro: cada opción describe una sección real del sitio.
